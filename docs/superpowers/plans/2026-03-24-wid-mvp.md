# wid MVP Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rust 製の `wid` CLI を立ち上げ、単一のグローバル Markdown ログに対して `wid` の全件表示と `wid now` の追記を行える MVP を実装する。

**Architecture:** `wid` は CLI 層、コマンド層、ログ操作層に分ける。保存の正本は `~/.local/share/wid/log.md` とし、`wid now` は append-only で当日セクションへ追記、`wid` は Markdown を読み取って表示する。対話入力は `wid now` 引数なし時の単一行プロンプトで扱う。

**Tech Stack:** Rust, `clap`, `anyhow`, 標準入出力, `cargo test`

---

## 実装方針メモ

- 初期版では YAGNI を徹底し、metadata や config は入れない
- `wid now` は shell 上で空白を含む入力をそのまま受けられるように、残り引数を空白連結する
- `wid now` 引数なし時は 1 行入力だけサポートする
- ログ既存内容はできるだけ再整形せず、必要最小限の追記だけ行う
- テストでは本物のホームディレクトリを書き換えないよう、ログパス解決に差し替えポイントを持たせる
